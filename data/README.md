# Synapse Data Preparation

This public release does not include datasets. You must download the preprocessed Synapse data yourself and place it under `data/Synapse/`.

Prepare Synapse in the common Swin-Unet / nnFormer format:

```text
data/Synapse/
├── train_npz/
│   ├── case0001_slice000.npz
│   ├── case0001_slice001.npz
│   ├── case0002_slice000.npz
│   └── ...
├── test_vol_h5/
│   ├── case0001.npy.h5
│   ├── case0002.npy.h5
│   └── ...
└── lists/lists_Synapse/
    ├── train.txt       # slice ids, e.g. case0031_slice000
    └── test_vol.txt    # case ids, e.g. case0008
```

File conventions:

- `train_npz/caseXXXX_sliceYYY.npz`
  - `image`: 2D CT slice
  - `label`: 2D segmentation mask
- `test_vol_h5/caseXXXX.npy.h5`
  - `image`: 3D CT volume with shape `(D, H, W)`
  - `label`: 3D segmentation mask with shape `(D, H, W)`
- `lists/lists_Synapse/train.txt`
  - one slice id per line, for example `case0031_slice000`
- `lists/lists_Synapse/test_vol.txt`
  - one case id per line, for example `case0008`

The TopoMamba-3D path automatically derives a case-level cache:

```text
data/Synapse/topomamba3d_nnunetlite/
├── manifests/
├── train_cases/
└── test_cases/
```

This cache is generated by:

```bash
python train_synapse.py --network TopoMamba_3D_t
```

or:

```bash
python test_synapse.py --network TopoMamba_3D_t --weights <checkpoint>
```

Use `SYNAPSE3D_REBUILD_CACHE=1` when you need to force regeneration.

The final 3D test follows `lists/lists_Synapse/test_vol.txt` exactly, so TopoMamba-2D and TopoMamba-3D use the same Synapse test case order.
